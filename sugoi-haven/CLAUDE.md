# CLAUDE.md — Sugoi Haven project briefing

This file briefs Claude Code (or any fresh Claude session) on this project. It encodes decisions made across the original design conversations. Read README.md as well — it has operational detail this file doesn't repeat.

## What this is

An e-commerce site for ukiyo-e Japanese woodblock prints (Hokusai's Thirty-Six Views of Mount Fuji, Hiroshige's Fifty-Three Stations of the Tōkaidō). Live on Netlify (site name "cheery-cactus-eac4da"), deploys automatically from the `main` branch of this GitHub repo. The owner is non-technical: explain changes plainly, avoid jargon in anything user-facing, and never assume they'll debug something themselves.

## The inviolable data-model rule

Three tiers: **Series** (a shop set) → **Design** (canonical artwork, exactly ONE page \+ ONE grid tile at `/shop/print/[slug]`, always) → **Listing** (one physical copy with its own publisher, condition, price, photos).

NEVER create a new Design/page/tile for a publisher or condition variant — those exist only as Listings compared side by side on the design's page. The inventory validator enforces this by rejecting unknown design\_ids.

## Data flow (do not hand-edit generated files)

- `src/data/designs.json` (55 Tōkaidō) — generated by `scripts/build-data.mjs` from the owner's seed CSV.  
- `src/data/fuji-designs.json` (46 Hokusai) — generated by `scripts/build-fuji-data.mjs`.  
- `src/data/listings.json` — generated by `scripts/fetch-inventory.mjs`, which runs as `prebuild`. Source of truth is the owner's **Google Sheet** (published CSV, `INVENTORY_CSV_URL` env var on Netlify). Locally with no env var it falls back to `scripts/demo-listings.json`. Validation is strict and fails builds loudly with Sheet row numbers — this is a feature, keep it.  
- `src/data/catalog.js` — the only hand-edited data file (Series definitions, helpers, artists, condition meanings).

## Build modes (automatic, don't hand-toggle)

- `npm run build` → deploy build: BrowserRouter, clean URLs, then a `postbuild` prerender (`scripts/prerender.mjs`) renders \~122 routes to static HTML with per-page SEO tags, JSON-LD, sitemap.xml, robots.txt, and a real `dist/404.html`.  
- `SINGLEFILE=1 npm run build` → one self-contained preview HTML (HashRouter); prerender auto-skips.  
- There is deliberately NO SPA catch-all redirect (`netlify.toml`): every real route is a prerendered file, unknown URLs must 404 for SEO. Do not re-add a `/* -> /index.html 200` rule.

## SEO system

`src/lib/seo.js` is the single source of truth for titles/descriptions/ JSON-LD, consumed at build time by `scripts/prerender.mjs` and at runtime by `src/lib/useSEO.js` (Layout). If you add a route, add it there too, or prerender will skip it. Cart/checkout/search/404 are noindex and excluded from the sitemap on purpose. `SITE_URL` env var switches the whole system to a custom domain when the owner buys one.

## Images

- Browse images are public-domain Wikimedia Commons reference scans, DEV PLACEHOLDERS ONLY, resolved by a candidate-fallback chain (`src/components/PrintImage.jsx`) with a localStorage cache (`sugoi-haven-img-cache-v1`) and an on-brand SVG placeholder floor.  
- `scripts/verify-images.mjs` (run on a machine with open internet) bakes the winning filename into the data — encourage the owner to run it.  
- Real Listing photos come from the Sheet's `photo_urls` column (pipe-separated URLs) and always override reference scans. Hard rule from the blueprint: a real listing must show photos of the exact physical sheet.

## Design system rules

Tokens in `src/index.css` (Tailwind v4 `@theme`): washi cream bg (never pure white), sumi ink text, aizuri indigo primary, bokashi gradient transitions. The warm `ume` accent is used ONLY for CTAs and prices — do not spread it. Fonts: Noto Serif JP (display) \+ Noto Sans JP (UI). Decorative motifs are original line art in `src/components/Marks.jsx` (never crop/copy the actual artworks for UI chrome). GSAP animations are subtle (\<0.8s), scroll-triggered via `[data-reveal]` \+ `usePageReveals` — any NEW page using DesignCard or `data-reveal` MUST call `usePageReveals()` or content renders invisible (this bug happened once; see PrintPage).

## Testing

`node scripts/smoke.mjs` after `npm run build` — jsdom-mounts \~19 routes against the deploy bundle (real paths, not hash URLs) and probes key content. Run it before pushing anything nontrivial. Note its limitation: it checks text presence, not visual rendering or opacity.

## What is mock / not yet built

- Checkout collects no payment (demo). Planned: Snipcart (fits qty-1 unique prints; auto-decrements stock). Not built yet.  
- Contact form and waitlist signups persist only to the visitor's own localStorage (`sugoi-haven-waitlist-v1`) — must be pointed at a real provider before they're trusted.  
- Remaining roadmap: custom domain \+ `SITE_URL` env var, Search Console sitemap submission, Snipcart checkout, owner's photography pass, phase-2 "compare copies" toggle on print pages.

## Workflow expectations

Commits to `main` auto-deploy to production via Netlify. Before pushing: build \+ smoke test locally. Explain every change to the owner in plain language. Prefer small, reviewable commits. The owner updates inventory via Google Sheet \+ a Netlify build-hook bookmark — never require them to edit code or JSON for routine inventory work.
